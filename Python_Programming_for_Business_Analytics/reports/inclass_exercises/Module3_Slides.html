Single vs Double Brackets

# Single brackets → Series
df['Age']

# Double brackets → DataFrame (even for one column)
df[['Age']]

Why does it matter?

  • Series: simpler, for single column operations
  • DataFrame: keeps table structure, needed for some operations

Selecting Rows by Position: iloc

iloc = integer location 
# Single row (returns Series)
df.iloc[0]       # First row
df.iloc[-1]      # Last row

# Multiple rows (returns DataFrame)
df.iloc[0:5]     # First 5 rows (0, 1, 2, 3, 4)
df.iloc[[0, 2, 4]]  # Specific rows

# Rows AND columns
df.iloc[0:5, 0:3]   # First 5 rows, first 3 columns
df.iloc[:, 0]       # All rows, first column

Selecting Rows by Label: loc

loc = location by label 
# By index label (default index is 0, 1, 2...)
df.loc[0]        # Row with index label 0
df.loc[0:4]      # Rows 0, 1, 2, 3, 4 (inclusive!)

# Rows and columns by name
df.loc[0:4, 'Name']              # Single column
df.loc[0:4, ['Name', 'Age']]     # Multiple columns
df.loc[0:4, 'Name':'Age']        # Column range
⚠️ Warning: loc includes the endpoint! loc[0:4] gives 5 rows. iloc excludes it like normal Python! iloc[0:4] gives 4 rows.

Boolean Filtering: The Concept

# Step 1: Create a condition (True/False for each row)
condition = df['Age'] >= 18
print(condition.head())

0    True
1    True
2    True
3    True
4    True
Name: Age, dtype: bool

# Step 2: Use condition to filter
adults = df[condition]
# Or in one line:
adults = df[df['Age'] >= 18]

Single Condition Examples

# Numeric comparisons
adults = df[df['Age'] >= 18]
children = df[df['Age'] < 18]
exact_age = df[df['Age'] == 30]

# Categorical comparisons
survivors = df[df['Survived'] == 1]
first_class = df[df['Pclass'] == 1]
women = df[df['Sex'] == 'female']

# Check results
print(f"Adults: {len(adults)}")
print(f"Survivors: {len(survivors)}")
print(f"Women: {len(women)}")

Comparison Operators

Operator Meaning Example
== Equal to df['Sex'] == 'female'
!= Not equal to df['Pclass'] != 3
< Less than df['Age'] < 18
> Greater than df['Fare'] > 100
<= Less than or equal df['Age'] <= 12
>= Greater than or equal df['Age'] >= 65

Multiple Conditions: AND

Use & (ampersand) for AND:

# Women who survived
women_survived = df[(df['Sex'] == 'female') & (df['Survived'] == 1)]
print(f"Women who survived: {len(women_survived)}")

# Adults in first class
adult_first = df[(df['Age'] >= 18) & (df['Pclass'] == 1)]
print(f"Adults in first class: {len(adult_first)}")

# Children who survived
child_survivors = df[(df['Age'] < 18) & (df['Survived'] == 1)]
print(f"Children who survived: {len(child_survivors)}")
⚠️ Warning: Important: Use & not and, and ALWAYS use parentheses around each condition!

Multiple Conditions: OR

Use | (pipe) for OR:

# First OR second class
upper_class = df[(df['Pclass'] == 1) | (df['Pclass'] == 2)]
print(f"Upper class: {len(upper_class)}")

# Very young or very old
extreme_ages = df[(df['Age'] < 5) | (df['Age'] > 70)]
print(f"Very young or old: {len(extreme_ages)}")

Negation: NOT

Use ~ (tilde) for NOT:

# NOT first class
not_first = df[~(df['Pclass'] == 1)]
print(f"Not first class: {len(not_first)}")

# NOT survived (same as Survived == 0)
died = df[~(df['Survived'] == 1)]
print(f"Did not survive: {len(died)}")

Complex Conditions

# Adult women in first class who survived
complex_filter = df[
    (df['Age'] >= 18) & 
    (df['Sex'] == 'female') & 
    (df['Pclass'] == 1) & 
    (df['Survived'] == 1)
]
print(f"Adult women in 1st class who survived: {len(complex_filter)}")

# Men OR children who survived
men_or_children_survived = df[
    ((df['Sex'] == 'male') | (df['Age'] < 18)) & 
    (df['Survived'] == 1)
]

Filtering with isin()

Check if value is in a list:

# Passengers in 1st or 2nd class
upper_class = df[df['Pclass'].isin([1, 2])]
print(f"1st or 2nd class: {len(upper_class)}")

# Embarked at Cherbourg or Southampton
ports = df[df['Embarked'].isin(['C', 'S'])]
print(f"Embarked at C or S: {len(ports)}")

# Specific ages
specific_ages = df[df['Age'].isin([25, 30, 35])]

Filtering with between()

Check if value is in a range:

# Age between 20 and 30 (inclusive)
young_adults = df[df['Age'].between(20, 30)]
print(f"Age 20-30: {len(young_adults)}")

# Fare between 10 and 50
mid_fare = df[df['Fare'].between(10, 50)]
print(f"Fare $10-$50: {len(mid_fare)}")

Why String Operations Matter

Real-world text data is messy:

  • Inconsistent capitalization: "Male", "MALE", "male"
  • Extra whitespace: " John "
  • Need to extract parts: "Mr. John Smith" → "Mr."
  • Need to search: Find names containing "Mrs"
Pandas provides .str accessor for string operations.

The .str Accessor

# Access string methods on a column
df['Name'].str.method_name()

# Examples:
df['Name'].str.lower()      # All lowercase
df['Name'].str.upper()      # All uppercase
df['Name'].str.len()        # Length of each string
df['Name'].str.strip()      # Remove whitespace

Case Conversion

# Create sample data
names = pd.Series(['  alice  ', 'BOB', 'Charlie', 'diana smith'])


print("Original:")
print(names.tolist())



print("\nLowercase:")
print(names.str.lower().tolist())



print("\nUppercase:")
print(names.str.upper().tolist())



print("\nTitle Case:")
print(names.str.title().tolist())



print("\nCapitalize (first letter only):")
print(names.str.capitalize().tolist())

Whitespace Handling

# Sample data with whitespace problems
dirty = pd.Series(['  Alice  ', 'Bob\t', '\nCharlie', '  Diana  '])

# Strip whitespace from both ends
clean = dirty.str.strip()
print(clean.tolist())  # ['Alice', 'Bob', 'Charlie', 'Diana']

# Strip only left side
dirty.str.lstrip()

# Strip only right side
dirty.str.rstrip()

Searching Text: contains()

# Find names containing "Mrs"
has_mrs = df['Name'].str.contains('Mrs')
mrs_passengers = df[has_mrs]
print(f"Mrs passengers: {len(mrs_passengers)}")

# Case-insensitive search
has_mrs = df['Name'].str.contains('mrs', case=False)

# Contains any of multiple patterns
has_title = df['Name'].str.contains('Mr|Mrs|Miss|Master')

More Search Methods

# Starts with
mr_names = df[df['Name'].str.startswith('Mr.')]

# df['Name'].str.match(pattern)

# Split names by comma

name_parts = df['Name'].str.split(',', expand=True)
name_parts.columns = ['LastName', 'FirstAndTitle']

# Simple replacement
df['Sex_short'] = df['Sex'].replace({'male': 'M', 'female': 'F'})

Why Dates Matter

Dates are everywhere in data:

  • Sales data: When was something purchased?
  • Events: When did something happen?
  • Time series: Stock prices, weather, website traffic
  • Age calculations: How old is a customer?
Dates are NOT strings! We need special handling.

Creating Date Data

# Sample data with dates as strings
data = {
    'event': ['Meeting', 'Conference', 'Workshop'],
    'date_str': ['2024-01-15', '2024-02-20', '2024-03-10']
}
df_dates = pd.DataFrame(data)

# Check the type - it's object (string)!
print(df_dates['date_str'].dtype)  # object

Converting Strings to Dates

# Convert to datetime
df_dates['date'] = pd.to_datetime(df_dates['date_str'])

# Now it's datetime64
print(df_dates['date'].dtype)  # datetime64[ns]
print(df_dates['date'])

0   2024-01-15
1   2024-02-20
2   2024-03-10
Name: date, dtype: datetime64[ns]

Handling Different Date Formats

# Pandas can often figure out the format
dates = ['01/15/2024', '02-20-2024', '2024.03.10', 'March 5, 2024']
pd.to_datetime(dates)

# Specify format explicitly for speed and accuracy
df['date'] = pd.to_datetime(df['date_str'], format='%Y-%m-%d')

# Common format codes:
# %Y = 4-digit year (2024)
# %y = 2-digit year (24)
# %m = month (01-12)
# %d = day (01-31)
# %H = hour (00-23)
# %M = minute (00-59)
# %S = second (00-59)

Date Format Codes

Code Meaning Example
%Y 4-digit year 2024
%y 2-digit year 24
%m Month (zero-padded) 01, 12
%d Day (zero-padded) 01, 31
%B Full month name January
%b Abbreviated month Jan
%A Full weekday name Monday
%a Abbreviated weekday Mon
%H Hour (24-hour) 00, 23
%I Hour (12-hour) 01, 12
%M Minute 00, 59
%S Second 00, 59

Handling Messy Date Formats

# Mixed formats - use format='mixed'
messy_dates = pd.Series(['2024-01-15', '01/20/2024', '2024.02.25'])
clean_dates = pd.to_datetime(messy_dates, format='mixed')

# Handle errors
# errors='coerce' converts bad dates to NaT (Not a Time)
dates_with_errors = pd.Series(['2024-01-15', 'invalid', '2024-03-10'])
clean = pd.to_datetime(dates_with_errors, errors='coerce')
print(clean)

0   2024-01-15
1          NaT
2   2024-03-10
dtype: datetime64[ns]

The .dt Accessor

Just like .str for strings, use .dt for datetime operations:

# Create sample dates
df_dates['date'] = pd.to_datetime(['2024-01-15', '2024-06-20', '2024-12-25'])

# Extract components
df_dates['year'] = df_dates['date'].dt.year
df_dates['month'] = df_dates['date'].dt.month
df_dates['day'] = df_dates['date'].dt.day
df_dates['weekday'] = df_dates['date'].dt.weekday  # 0=Monday, 6=Sunday
df_dates['day_name'] = df_dates['date'].dt.day_name()
df_dates['month_name'] = df_dates['date'].dt.month_name()


print(df_dates)

Calculating Age from Birthdate

# Sample birthdate data

    'name': ['Alice', 'Bob', 'Charlie'],
    'birthdate': pd.to_datetime(['1990-05-15', '1985-12-20', '2000-03-10'])
})

# Calculate age
today = pd.Timestamp.today()
people['age'] = (today - people['birthdate']).dt.days // 365

# More accurate age calculation
people['age_accurate'] = (
    today.year - people['birthdate'].dt.year - 
    ((today.month < people['birthdate'].dt.month) | 
     ((today.month == people['birthdate'].dt.month) & 
      (today.day < people['birthdate'].dt.day)))
)

Part 7: GroupBy Operations

Split-Apply-Combine (30 minutes)

What is GroupBy?

GroupBy follows the split-apply-combine pattern:

  1. Split: Divide data into groups
  2. Apply: Calculate something for each group
  3. Combine: Put results back together

Basic GroupBy Syntax

# Average fare by passenger class
avg_fare = df.groupby('Pclass')['Fare'].mean()
print(avg_fare)

Pclass
1    84.154687
2    20.662183
3    13.675550
Name: Fare, dtype: float64

GroupBy Structure

# Syntax:
df.groupby('column_to_group_by')['column_to_aggregate'].function()

# Examples:
df.groupby('Sex')['Survived'].mean()      # Survival rate by sex
df.groupby('Pclass')['Age'].mean()        # Average age by class
df.groupby('Embarked')['Fare'].sum()      # Total fare by port

Common Aggregation Functions

Function Description
mean() Average
sum() Total
count() Number of values
min() Minimum
max() Maximum
std() Standard deviation
median() Middle value
first() First value
last() Last value
nunique() Number of unique values

Multiple Aggregations with agg()

# Multiple functions on one column
df.groupby('Pclass')['Fare'].agg(['mean', 'min', 'max', 'count'])

# Different functions for different columns
summary = df.groupby('Pclass').agg({
    'Survived': 'mean',           # Survival rate

    'Fare': 'mean',               # Average fare
    'PassengerId': 'count'        # Count

Filling Missing Values

# Fill with a specific value
df['Cabin_filled'] = df['Cabin'].fillna('Unknown')

# Fill with mean (for numeric columns)
df['Age_filled'] = df['Age'].fillna(df['Age'].mean())

# Fill with median (better for skewed data)
df['Age_filled'] = df['Age'].fillna(df['Age'].median())

# Fill with mode (for categorical)
df['Embarked'] = df['Embarked'].fillna(df['Embarked'].mode()[0])

Dropping Missing Values

# Drop rows with ANY missing value
df_clean = df.dropna()
print(f"Rows remaining: {len(df_clean)}")  # Only 183!

# Drop rows missing specific column
df_with_age = df.dropna(subset=['Age'])
print(f"Rows with age: {len(df_with_age)}")  # 714

# Drop columns with too many missing
df_less_cols = df.dropna(axis=1, thresh=500)  # Keep cols with >= 500 non-null
