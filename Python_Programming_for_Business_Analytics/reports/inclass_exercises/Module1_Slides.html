Part 2: Variables

Storing and Naming Your Data (45 minutes)

What is a Variable?

A variable is like a labeled box that stores data.

name = "Alice"
  • name is the label (variable name)
  • "Alice" is the content (value)
  • = is the assignment operator (puts value in box)
Think of it as: "Create a box labeled 'name' and put 'Alice' inside"

Why Do We Need Variables?

Without variables: 

print(5 + 3)
print(5 * 3)
print(5 - 3)
# What if we want to change 5 to 10? Must change everywhere!

With variables: 

number = 5
print(number + 3)
print(number * 3)
print(number - 3)
# To change, just modify once: number = 10

Variables make code flexible and reusable.

Creating Variables

# Text (called a "string")
name = "Alice"

# Whole number (called an "integer")
age = 28

# Decimal number (called a "float")
height = 1.65

# True/False value (called a "boolean")
is_student = True
📝 Note: Lines starting with # are comments - Python ignores them. Use them to explain your code!

Using Variables

Once created, use variables by their name:

name = "Alice"
age = 28


print(name)
print(age)
print(name)  # Can use as many times as you want

Output: 
Alice
28
Alice

Variables remember their values until you change them or the program ends.

Changing Variable Values

Variables can change (that's why they're called "variables"!):

score = 100
print(score)


score = 150
print(score)



score = score + 50  # Use old value to calculate new value
print(score)

Output: 
100
150
200

Understanding Assignment

The = sign in Python is NOT like math equality!

x = 5      # "Put 5 into the box labeled x"
x = x + 1  # "Take value from x, add 1, put result back in x"
print(x)   # 6

This is different from math where x = x + 1 makes no sense!

In Python, = means "assign the value on the right to the variable on the left."

Variable Naming Rules (MUST Follow)

Required rules:
  1. Must start with letter or underscore (_)
  2. Can only contain letters, numbers, underscores
  3. Cannot contain spaces or special characters
  4. Cannot be Python keywords
  5. Case-sensitive (age ≠ Age ≠ AGE)
# VALID names:
name = "Alice"
first_name = "Alice"
name2 = "Alice"
_private = "Alice"

# INVALID names (will cause errors):
# 2name = "Alice"      # Starts with number
# first-name = "Alice" # Contains hyphen
# first name = "Alice" # Contains space
# class = "Alice"      # Python keyword

Python Keywords (Reserved)

These words have special meaning - you can't use them as variable names:

False    class     finally   is        return
None     continue  for       lambda    try
True     def       from      nonlocal  while
and      del       global    not       with
as       elif      if        or        yield
assert   else      import    pass
break    except    in        raise

If you try to use one, you'll get a SyntaxError.

Variable Naming Conventions (SHOULD Follow)

Python Style Guide (PEP 8) recommendations: 
# GOOD - snake_case (words separated by underscores)
student_name = "Alice"
total_score = 95
is_valid = True

# AVOID - but technically valid
studentName = "Alice"   # camelCase (used in JavaScript)
StudentName = "Alice"   # PascalCase (used for classes)
TOTAL_SCORE = 95        # SCREAMING_CASE (used for constants)

# BAD - hard to read
studentname = "Alice"
s = "Alice"
x = 95

Descriptive Variable Names

# BAD - What do these mean?
a = 50000
b = 0.22
c = a - (a * b)

# GOOD - Self-documenting code
salary = 50000
tax_rate = 0.22
net_income = salary - (salary * tax_rate)

Good variable names make code readable without needing comments.

Float Precision Warning

False

This is a fundamental limitation of how computers store decimals. For money, consider using integers (cents) or the decimal module.

Order of Operations (PEMDAS/BODMAS)

Python follows standard math order:

  1. Parentheses / Brackets
  2. Exponents / Orders
  3. Multiplication and Division (left to right)
  4. Addition and Subtraction (left to right)
print(2 + 3 * 4)       # 14 (not 20!)
print((2 + 3) * 4)     # 20
print(2  3  2)     # 512 (exponents: right to left)
print((2  3)  2)   # 64
print(10 - 4 - 2)      # 4 (left to right)

Assignment Operators (Shortcuts)

Instead of x = x + 5, use shortcuts:

x = 10

print(f"After += 5: {x}")

print(f"After -= 3: {x}")



x = 2   # Same as: x = x  2
print(f"After *= 2: {x}")



x //= 4  # Same as: x = x // 4
print(f"After //= 4: {x}")

String Methods

More String Methods

text = "  Hello, World!  "

# Whitespace handling
print(text.strip())       # "Hello, World!" (removes both sides)
print(text.lstrip())      # "Hello, World!  " (left only)
print(text.rstrip())      # "  Hello, World!" (right only)

# Finding and replacing
print(text.strip().replace("World", "Python"))  # Hello, Python!
print(text.strip().count("l"))                  # 3
print(text.strip().find("World"))               # 7 (index)

String Checking Methods

# Check what string contains
print("hello".isalpha())      # True (only letters)
print("12345".isdigit())      # True (only digits)
print("hello123".isalnum())   # True (letters or digits)
print("   ".isspace())        # True (only whitespace)

# Check start/end
text = "Hello, World!"
print(text.startswith("Hello"))  # True
print(text.startswith("hello"))  # False (case-sensitive!)
print(text.endswith("!"))        # True
print(text.endswith("?"))        # False

String Indexing

Access individual characters by position (index):

word = "Python"
#       012345  (positive index, from left)
#      -654321  (negative index, from right)


print(word[0])    # P (first)
print(word[1])    # y (second)
print(word[5])    # n (sixth, last)
print(word[-1])   # n (last)
print(word[-2])   # o (second to last)

⚠️ Warning: Indexing starts at 0, not 1! First character is [0].

String Slicing

Extract substrings with [start:stop:step]:

word = "Python"


print(word[0:3])   # Pyt (index 0, 1, 2)
print(word[:3])    # Pyt (from start to 3)
print(word[3:])    # hon (from 3 to end)
print(word[1:5])   # ytho (1, 2, 3, 4)
print(word[::2])   # Pto (every 2nd character)
print(word[::-1])  # nohtyP (reversed!)

Note: The stop index is NOT included (exclusive).

Part 5: F-Strings

Formatted String Literals (20 minutes)

The Problem

How do we combine text and variables?

name = "Alice"
age = 28

# Concatenation is clunky:
message = "My name is " + name + " and I am " + str(age) + " years old."
print(message)

There must be a better way!

F-Strings to the Rescue!

Put f before the string and variables inside {curly braces}:

name = "Alice"
age = 28

# F-string way - clean and readable!
message = f"My name is {name} and I am {age} years old."
print(message)
Output: 
My name is Alice and I am 28 years old.

F-String Syntax

name = "Alice"
score = 95

# The f before quotes is required!
greeting = f"Hello, {name}!"
result = f"You scored {score} points."


print(greeting)
print(result)


# Without the f, braces are literal text:
no_f = "Hello, {name}!"  # Prints: Hello, {name}!
print(no_f)

Expressions in F-Strings

You can put any Python expression inside the braces:

x = 10
y = 5


print(f"Sum: {x + y}")
print(f"Product: {x * y}")
print(f"x squared: {x ** 2}")
print(f"Is x bigger? {x > y}")
print(f"Division: {x / y}")

Output: 
Sum: 15
Product: 50
x squared: 100
Is x bigger? True
Division: 2.0

Function Calls in F-Strings

name = "alice"
items = ["apple", "banana", "cherry"]


print(f"Original: {name}")
print(f"Upper: {name.upper()}")
print(f"Title: {name.title()}")
print(f"Length: {len(name)}")
print(f"Number of items: {len(items)}")

Output: 
Original: alice
Upper: ALICE
Title: Alice
Length: 5
Number of items: 3

Number Formatting: Decimal Places

pi = 3.14159265359
price = 19.5

# .Xf = X decimal places
print(f"Pi: {pi}")           # 3.14159265359
print(f"Pi: {pi:.2f}")       # 3.14
print(f"Pi: {pi:.4f}")       # 3.1416


print(f"Price: ${price}")    # $19.5
print(f"Price: ${price:.2f}")# $19.50

The f after the number means "fixed-point" (show as decimal).

Number Formatting: Thousands Separator

population = 2100000
salary = 52750.50

# Use comma for thousands separator
print(f"Population: {population:,}")
print(f"Salary: ${salary:,.2f}")

# Use underscore (Python 3.6+)
print(f"Population: {population:_}")
Output: 
Population: 2,100,000
Salary: $52,750.50
Population: 2_100_000

Number Formatting: Percentages

ratio = 0.756
accuracy = 0.9823
completion = 0.5


print(f"Ratio: {ratio:.1%}")       # 75.6%
print(f"Accuracy: {accuracy:.2%}") # 98.23%
print(f"Complete: {completion:.0%}")# 50%

The % multiplies by 100 and adds the percent sign.

Number Formatting: Alignment and Padding

name = "Alice"
score = 42

# Width and alignment
print(f"|{name:>15}|")  # Right align, width 15
print(f"|{name:<15}|")  # Left align
print(f"|{name:^15}|")  # Center

# Zero padding
print(f"ID: {score:05}")    # 00042
print(f"Hex: {score:04x}")  # 002a
Output: 

  

    Alice
  

  

    Alice
  

  

    Alice
  


ID: 00042
Hex: 002a

Part 6: Common Errors

Understanding and Fixing Mistakes (20 minutes)

Errors Are Normal!

  • Every programmer makes errors - lots of them!
  • Errors are how you learn
  • Python gives helpful error messages
  • Reading errors is a critical skill
Don't be afraid of errors - learn from them!

Anatomy of an Error Message

Traceback (most recent call last):
  File "script.py", line 3, in 
    print(nmae)
NameError: name 'nmae' is not defined
  1. Traceback - The path to the error
  2. File and line - Exact location
  3. Code - The line that caused the error
  4. Error type - What kind of error (NameError)
  5. Message - Description of the problem
Always read the LAST line first!

NameError

Cause: Using a variable that doesn't exist (usually a typo) 
name = "Alice"
print(nmae)  # Typo!
Error: 
NameError: name 'nmae' is not defined
Fix: Check spelling carefully! 
name = "Alice"
print(name)  # Correct

TypeError

Cause: Using wrong type for an operation 
age = 25
message = "I am " + age + " years old"  # Can't add string + int!
Error: 
TypeError: can only concatenate str (not "int") to str
Fix: Convert types or use f-string! 
message = "I am " + str(age) + " years old"  # Convert to string
message = f"I am {age} years old"            # Or use f-string

SyntaxError

Cause: Invalid Python syntax 
print("Hello World)  # Missing closing quote
Error: 
SyntaxError: EOL while scanning string literal
Common causes:
  • Missing quotes, parentheses, brackets
  • Unmatched brackets/quotes
  • Missing colons after if/for/while
  • Wrong indentation

More SyntaxErrors

# Missing colon
if True
    print("yes")

# Mismatched brackets
data = [1, 2, 3)

# Invalid assignment
5 = x

# Keyword as variable
class = "Math"

ValueError

Cause: Right type, wrong value 
number = int("hello")  # Can't convert "hello" to int
Error: 
ValueError: invalid literal for int() with base 10: 'hello'
Fix: Validate input before converting 
text = "hello"
if text.isdigit():
    number = int(text)
else:
    print("Not a valid number!")

IndexError

Cause: Accessing an index that doesn't exist 
word = "Python"  # Indices 0-5
print(word[10])  # Index 10 doesn't exist!
Error: 
IndexError: string index out of range
Fix: Check length first 
word = "Python"
index = 10
if index < len(word):
    print(word[index])
else:
    print(f"Index {index} is out of range!")

ZeroDivisionError

Cause: Dividing by zero 
result = 10 / 0
Error: 
ZeroDivisionError: division by zero
Fix: Check before dividing 
numerator = 10
denominator = 0


if denominator != 0:
    result = numerator / denominator
else:
    print("Cannot divide by zero!")

AttributeError

Cause: Using method/attribute that doesn't exist for that type 
number = 42
print(number.upper())  # Numbers don't have .upper()!
Error: 
AttributeError: 'int' object has no attribute 'upper'
Fix: Use correct method for the type 
text = "hello"
print(text.upper())  # Strings have .upper()
